# 테크노트 — 얼굴·차량번호판 자동 블러: 개발 전 스킬 갈무리

다음 개발 목표는 **사진 속 얼굴·차량번호판을 자동으로 가리는(블러) 기능**이다. 본격 구현 전에 쓸 만한 기술·스킬을 한 번 정리해두려고 한다. (우리 명세 FR-008 / SC-009 "비식별 처리율 100%"에 해당)

- **요구사항**: 시민이 올린 사진에서 ① 사람 얼굴 ② 차량번호판을 찾아 ③ 가린(블러/모자이크) 사본만 저장. 원본은 저장하지 않는다.
- **우리 원칙(중요)**: 가능하면 **업로드 전에 브라우저(기기)에서** 블러 → 원본이 서버로 아예 안 나간다. 키·비용 0원 기조 유지(현재 분류와 동일).
- 현재 코드에선 `anonymizePhoto()`가 플래그만 세우는 **스텁** → 이걸 진짜 블러 파이프라인으로 교체하는 게 이번 작업.

---

## 1. 후보 기술 비교 (얼굴 / 번호판 / 블러)

| 대상 | 브라우저(무료·무키) 후보 | 서버·고정확 후보 |
|------|--------------------------|------------------|
| **얼굴 탐지** | ★ MediaPipe Face Detector(BlazeFace) · face-api.js · 네이티브 FaceDetector(실험적·불안정) | RetinaFace · MediaPipe · OpenCV DNN |
| **번호판 탐지** | ★ OWL-ViT 제로샷(Transformers.js, "license plate" 질의) · OpenCV.js Haar(plate cascade) · 텍스트영역(EAST) | YOLOv8 번호판 전용 모델(ultralytics) |
| **가리기(블러)** | ★ Canvas `filter:blur()` / 픽셀화 | OpenCV GaussianBlur/모자이크 |

★ = 우리 스택(브라우저 우선·무키)에 가장 잘 맞는 1순위.

### 핵심 포인트
- **얼굴**: MediaPipe `tasks-vision`의 Face Detector가 사실상 정답. 빠르고 정확하고 CDN으로 바로 로드, 모바일에서도 잘 돈다.
- **번호판**: 브라우저에 "번호판 전용" 표준 모델이 없다. 그래서 **OWL-ViT 제로샷 객체탐지**(텍스트로 "license plate"를 질의하면 박스를 찍어줌)가 키 없이 쓸 수 있는 현실적 카드. 단 **무겁고 느림**(주의 섹션 참고).
- **블러**: 탐지된 박스 영역만 캔버스에서 흐리거나 픽셀화. **되돌릴 수 없게 픽셀화 권장**(모자이크 복원 공격 방지).

## 2. 추천 조합 (1차 구현안)

> **브라우저 온디바이스 파이프라인** — 사진 첨부 → 얼굴+번호판 탐지 → 박스 블러 → 블러본만 분류·업로드.

1. 얼굴: **MediaPipe Face Detector**
2. 번호판: **OWL-ViT 제로샷**(너무 느리면 → 서버 YOLO로 승급, 또는 OpenCV.js Haar 경량 폴백)
3. 블러: **Canvas 픽셀화**
4. 결과 dataURL을 기존 `photoDataUrl`로 사용(분류·저장은 블러본 기준)

정확도를 강하게 요구하면(번호판 측면·야간) **서버 Python(YOLOv8 번호판 + MediaPipe/RetinaFace + OpenCV 블러)** 로 가는 2차안을 둔다. (Claude 비전처럼 키 필요/비용 발생)

## 3. 구현 스케치 (붙일 코드 자리: `vision.js`의 비식별 단계)

### 얼굴 — MediaPipe tasks-vision
```js
import { FilesetResolver, FaceDetector } from 'https://cdn.jsdelivr.net/npm/@mediapipe/tasks-vision';
const fileset = await FilesetResolver.forVisionTasks('https://cdn.jsdelivr.net/npm/@mediapipe/tasks-vision/wasm');
const faceDetector = await FaceDetector.createFromOptions(fileset, {
  baseOptions: { modelAssetPath: 'https://storage.googleapis.com/mediapipe-models/face_detector/blaze_face_short_range/float16/1/blaze_face_short_range.tflite' },
  runningMode: 'IMAGE',
});
const faces = faceDetector.detect(imgEl).detections
  .map(d => d.boundingBox); // {originX, originY, width, height}
```

### 번호판 — OWL-ViT 제로샷 (Transformers.js)
```js
import { pipeline } from 'https://cdn.jsdelivr.net/npm/@huggingface/transformers';
const detect = await pipeline('zero-shot-object-detection', 'Xenova/owlvit-base-patch32');
const plates = (await detect(dataUrl, ['a license plate', 'car number plate'], { threshold: 0.1 }))
  .map(o => o.box); // {xmin, ymin, xmax, ymax}
```

### 블러(픽셀화) — Canvas
```js
function anonymize(img, boxes) {
  const c = document.createElement('canvas');
  c.width = img.naturalWidth; c.height = img.naturalHeight;
  const ctx = c.getContext('2d'); ctx.drawImage(img, 0, 0);
  for (const b of boxes) {
    const x=b.x, y=b.y, w=b.w, h=b.h, s=Math.max(4, Math.floor(w/12));
    // 영역을 작게 줄였다가 키워서 모자이크(비가역)
    ctx.imageSmoothingEnabled = false;
    ctx.drawImage(c, x, y, w, h, x, y, Math.ceil(w/s), Math.ceil(h/s));
    ctx.drawImage(c, x, y, Math.ceil(w/s), Math.ceil(h/s), x, y, w, h);
  }
  return c.toDataURL('image/jpeg', 0.85);
}
```
> 박스 좌표계(원본 픽셀 기준)와 다운스케일본 좌표계가 다를 수 있으니, **원본에서 탐지·블러 후** 마지막에 다운스케일하는 순서로 맞춘다.

### 흐름 교체
- `onPhoto()`에서: 원본 로드 → (얼굴+번호판 탐지) → `anonymize()` → 그 결과를 `CivicVision.downscale` → 분류·업로드. `pii_removed = (faces.length+plates.length>0)` 또는 항상 true.

## 5. 주의점 (구현 전에 합의해둘 것)

- **정확도 한계**: 브라우저 모델은 측면·저화질·야간 번호판/얼굴을 놓칠 수 있다. → "100% 못 가림"을 전제로 **안전망**(관제측 2차 블러/수동 마스킹, 못 가린 신고 검토 큐)을 둔다.
- **성능**: OWL-ViT는 **무겁다(로딩 수백 MB·추론 수 초)**. 모바일 UX 고려 — 최초 1회 로딩 안내, WebGPU 활용, 너무 느리면 번호판은 서버로.
- **비가역 처리**: 단순 blur는 복원 시도가 가능 → **픽셀화(모자이크)** 로 비가역 처리.
- **순서**: 반드시 **업로드/저장 전에** 블러. 원본 dataURL은 메모리에서만 쓰고 폐기, **블러본만** 저장(`photo_url`).
- **법·정책**: 개인정보보호법 — 비식별 100% 목표(SC-009). 초상권·번호판은 개인정보. 못 가린 경우의 파기/재처리 절차 명시.

## 6. 다음 단계 (개발 착수 순서)

1. `vision.js`에 **얼굴 탐지(MediaPipe)** 먼저 붙이고 블러 → 가장 임팩트 크고 안정적.
2. **번호판(OWL-ViT)** 추가, 성능 측정 → 느리면 OpenCV.js 폴백 또는 서버 승급 결정.
3. `anonymizePhoto` 스텁 제거, **블러본만 저장**되도록 흐름 교체 + `pii_removed` 실제화.
4. 관제 안전망(못 가린 케이스) UI 추가.
5. 양쪽 채널(Pages·터널) 반영 + 백업 태그.

> 결론: **얼굴은 MediaPipe로 바로 가고**, 번호판은 **OWL-ViT 제로샷으로 시작하되 무거우면 서버 YOLO로 승급**하는 2단계 전략. 블러는 캔버스 픽셀화로 비가역 처리. 전부 **업로드 전 온디바이스** 가 핵심.
