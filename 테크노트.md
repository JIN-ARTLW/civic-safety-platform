# 사진 한 장으로 도시 위험을 신고하는 플랫폼, AI로 만들어봤습니다 (feat. 키 0원)

> 비즈니스 모델 캔버스 한 장에서 시작해서, 실제로 접속 가능한 데모까지 만든 기록.
> 스마트시티 안전 신고 플랫폼 MVP 개발기 · 2026.06

---

## 들어가며 — 왜 만들었나

"시민이 길에서 포트홀을 봤을 때, **사진 한 장만 찍으면 신고가 끝나면 어떨까?**"

요즘 안전신문고로 신고하려면 분야 고르고, 제목 쓰고, 내용 쓰고, 위치 찍고… 생각보다 손이 많이 갑니다. 그래서 *"사진을 AI가 보고 알아서 분류하고, 신고서까지 채워주는"* 플랫폼을 만들어보기로 했습니다.

목표는 두 가지였어요.
- **시민**: 텍스트 타이핑 없는 "3초 신고"
- **지자체**: 자동 분류·우선순위로 처리 시간 단축

시작은 거창하게 **비즈니스 모델 캔버스 한 장**이었습니다. 이걸 실제 동작하는 서비스로 어디까지 끌고 갈 수 있을지가 이번 프로젝트의 도전이었습니다.

## 1. 명세부터 — Spec-Kit으로 뼈대 잡기

바로 코딩하지 않고 GitHub **Spec Kit**으로 명세→계획→작업을 먼저 정리했습니다.

```
/speckit-specify   → 기능 명세 (사용자 스토리, 요구사항)
/speckit-clarify   → 애매한 부분 질문으로 확정 (신원모델, 멀티테넌시, 보관기간…)
/speckit-plan      → 기술 계획
/speckit-tasks     → 78개 작업으로 분해 (프론트/백엔드/도메인 3요소 명시)
/speckit-analyze   → 누락·불일치 점검
/speckit-implement → 구현
```

이 단계에서 좋았던 건, **"익명 신고 vs 로그인 필수"** 같은 핵심 결정을 코드 짜기 전에 못 박아둔 것. 나중에 뒤집으면 엄청 아팠을 부분이었습니다.

## 2. 첫 번째 현실 자각 — "풀스택 다 못 올려요"

계획서엔 멋지게 적혀 있었습니다. *Next.js + Supabase(PostgreSQL+PostGIS+pgvector) + Python AI 서비스*.

근데 막상 환경에서 이걸 **검증 가능하게** 다 띄우려니 무리였어요. 그래서 결정했습니다.

> **"인프라는 흉내내더라도, 도메인 로직·API·UX·AI 분류는 진짜로 돌게 만들자."**

그래서 의존성 없는 **Node 내장 모듈**로 백엔드를, 정적 HTML/JS로 프론트를 만들었습니다. 데이터는 JSON 파일, 멀티테넌트 격리는 코드 레벨로. 핵심 흐름은 진짜로 동작하되, 관리형 DB·실 LLM은 "후속 전환"으로 솔직하게 분리했습니다.

이 선택 덕분에 `npm install` 한 번 없이 `node server.mjs`로 바로 뜨는 가벼운 MVP가 나왔습니다.

## 3. AI 분류, 세 번 갈아엎은 이야기 🔄

여기가 이번 프로젝트 제일 재밌었던(=삽질한) 부분입니다.

### 시도 1: 파일명 키워드 매칭 (스텁)
처음엔 그냥 파일명에 "pothole" 들어있으면 도로파손, 이런 식. 당연히 **가짜**죠. 데모는 되는데 양심에 찔립니다.

### 시도 2: 브라우저에서 진짜 객체 탐지 (COCO-SSD)
TensorFlow.js로 **브라우저에서 실제 비전 모델**을 돌렸습니다. 키도 서버도 필요 없이 사진을 진짜로 분석! …근데 함정이 있었어요.

> 사용자가 **누가 봐도 포트홀**인 사진을 올렸는데 → "객체 없음, 미정 30%" 😇

COCO-SSD는 사람·차·신호등 같은 **일반 사물 80종**만 압니다. '포트홀'이라는 클래스가 아예 없어서, 차도 사람도 없는 노면 클로즈업은 그냥 못 알아봐요. 위험의 **"상태(파손됐나·침수됐나)"** 는 이해를 못 하는 거죠.

### 시도 3: 제로샷 분류 (CLIP) — 정답
그래서 **CLIP 제로샷 분류**(Transformers.js)로 갈아탔습니다. 이건 우리가 원하는 라벨을 **글로 정의**하면, 모델이 사진을 그 문장들과 의미적으로 대조해서 점수를 매깁니다.

```js
// 우리가 원하는 위험을 "문장"으로 정의
"a pothole or cracked damaged asphalt road surface" → 도로 파손
"a flooded street with standing water"              → 침수
"a pile of illegally dumped garbage bags"           → 무단투기
"a car illegally parked on the street"              → 불법주정차
```

키 없이, 브라우저에서, **포트홀/침수/쓰레기/불법주정차를 의미로 구분**합니다. COCO의 한계를 근본적으로 푼 순간이었어요. (보험으로 노면 손상 픽셀 분석 휴리스틱도 폴백으로 넣어뒀습니다.)

> 💡 교훈: "객체를 탐지"하는 모델과 "상황을 이해"하는 모델은 다르다. 후자가 필요하면 제로샷/VLM으로 가야 한다.

그리고 더 정확하게 가고 싶은 사람을 위해, **Claude 비전 API**를 키만 넣으면 자동 승급되도록 백엔드에 코드도 심어뒀습니다. (단, 키는 서버에만. Pages 같은 정적 호스팅엔 절대 안 둠 — 노출되니까.)

## 4. 위치 입력 — 사진이 다 알고 있었다

위험 신고에서 위치는 필수인데, 시민한테 위/경도 치라고 할 순 없잖아요. 그래서 3단 콤보로:

1. **사진 EXIF에서 GPS 자동 추출** (`exifr`) — 폰으로 찍은 사진엔 보통 위치가 박혀 있습니다. 첨부만 하면 끝.
2. 없으면 **현재 위치** (브라우저 위치 권한)
3. 그것도 안 되면 **한국 주소 검색** — 다음(카카오) 우편번호 팝업 + 무료 지오코딩(Nominatim)으로 좌표 변환

여기서도 삽질 하나. 처음에 넣은 다음 우편번호 스크립트 URL이 **폐기돼서 403**이 뜨더라고요. `index.v1/main.js` → `postcode.v2.js`로 바꾸니 해결. (옛날 블로그 보고 복붙하면 이런 일이…)

그리고 위/경도 입력칸은 아예 **숨겼습니다.** 시민은 좌표 볼 필요 없으니까. 내부적으로만 잡고, 화면엔 "📷 사진 GPS에서 위치 추출됨 (36.6424, 127.489)"처럼 사람이 읽을 정보만.

## 5. 진짜 안전신문고처럼 — 3대 분류와 양식 자동작성

실제 안전신문고를 보니 신고가 **3대 분야**로 나뉘고, 분야마다 양식이 조금씩 달랐습니다.

- **자동차·교통 위반** (차량번호 칸이 따로 있음)
- **생활불편** (쓰레기, 불법광고물, 자전거 방치…)
- **안전** (도로·시설물, 침수, 대기/수질오염, 소방…)

그래서 이렇게 만들었어요.
> 사진 분석 → **세부유형 판별 → 소속 분야 자동 결정 → 그 분야 양식 표시 → AI가 제목·내용·유형·일시를 자동으로 채움**

교통 분야면 차량번호 칸이 뜨고, 발생 일시는 **사진 촬영 시각(EXIF)** 을 우선 넣되 없으면 비워둡니다(직접 입력 가능). 시민이 할 일은 *"확인하고 제출"* 뿐. 개인정보(인적사항)는 이번 범위에선 일부러 뺐습니다.

지자체 관제 대시보드엔 이게 다 정리돼서 넘어갑니다 — 사진 썸네일, 분야·유형, 제목, 차량번호, 일시, **AI가 뭐라고 판단했는지 요약**까지.

## 6. 배포 — "그래서 어디서 보나요?"

만들었으면 남이 봐야죠. 근데 도메인(`p3.sumzip.com`)이 제 소유 장비가 아니라 못 쓰는 상황. 그래서 두 갈래로 풀었습니다.

### ① GitHub Pages (상시, 정적)
프론트를 정적 버전으로 만들어 Pages에 올림. 백엔드 로직은 **브라우저로 포팅**(localStorage 저장)해서 키·서버 없이 완전 동작. 항상 켜져 있고 영구적.

### ② cloudflared 터널 (풀스택, 공유 백엔드)
진짜 공유 서버가 필요한 버전은 **cloudflared 무료 터널**로 이 컴퓨터를 그대로 외부 공개. 계정도 도메인도 필요 없습니다.

```bash
cloudflared tunnel --url http://localhost:9500   # 끝. 공개 URL 발급
```

여기서 502 Bad Gateway도 한 번 만났는데, 알고 보니 터널은 살아있고 **그 뒤의 게이트웨이가 죽어서** 전달할 곳이 없던 거였어요. 그래서 아예 **킬스위치 스크립트**를 만들었습니다.

```bash
./tunnel.sh on       # 백엔드+게이트웨이 자동 점검 후 터널 켜기
./tunnel.sh off      # 터널만 끄기
./tunnel.sh status   # 상태 한눈에
```

> ⚠️ 무료 터널은 켤 때마다 주소가 바뀝니다. 데모할 땐 이게 좀 불편한데, 고정 주소 원하면 Cloudflare 계정 연결이나 클라우드 배포로 가야 합니다.

## 7. 버전 백업 — 중간발표 직전의 보험

중간발표를 앞두고 *"지금 잘 되는 이 상태"* 를 박제하고 싶었습니다. 그래서:

```bash
git tag -a v0.1-midterm -m "중간발표 버전 백업"
git branch backup/midterm-2026-06-19
```

그리고 발표 때 보여줄 용으로, 이 백업 버전의 정적 파일을 `docs/backup-v0.1/` 폴더에 **고정 복사**해 뒀습니다. 메인을 계속 고쳐도 이 폴더는 안 바뀌니까, **"중간발표 시점 그대로 볼 수 있는 페이지"** 가 생긴 거죠. 라이브 개발 링크 2개는 그대로 두고요.

## 정직하게 — 되는 것 / 안 되는 것

데모가 그럴듯하다고 다 된 게 아니라서, 솔직하게 적어둡니다.

**✅ 진짜 됩니다**: 사진 신고, AI 위험 분류(보이는 위험), 분야별 신고서 AI 자동작성, 위치 3종, 접수·상태조회, 관할 판별·중복묶음, 관제 처리·공문서 초안, 배포.

**⚠️ 흉내만**: 공문서 자동작성(값 채우기 수준), 분류 정확도(신호위반·과속처럼 사진 한 장으로 알기 어려운 건 약함→사용자가 수정).

**❌ 아직**: 대화형 챗봇(정해진 문구만), 얼굴·번호판 자동 가리기, 처리 알림(문자/푸시), 실제 안전신문고 자동 전송, 허위·도배 차단. 그리고 공공으로 가려면 **CSAP 인증 클라우드·ISMS-P·개인정보 영향평가**라는 큰 관문이 남아 있습니다.

## 마치며 — 캔버스 한 장에서 여기까지

이번 프로젝트로 배운 것:

1. **명세를 먼저.** 코딩 전에 결정을 못 박으면 나중이 편하다.
2. **"진짜 모델"과 "진짜처럼 보이는 것"은 다르다.** COCO에서 CLIP으로 갈아탄 게 제일 큰 분기점.
3. **무료로 멀리 갈 수 있다.** 브라우저 ML + GitHub Pages + cloudflared 조합이면 키·비용 0원으로 공개 데모까지.
4. **솔직함이 신뢰.** 되는 것과 안 되는 것을 구분해 적는 게, 화려한 데모보다 낫다.

비즈니스 모델 캔버스 한 장이 *"사진 찍어 신고 → AI 분류 → 자동 신고서 → 지자체 처리"* 가 실제로 돌아가는 데모가 됐습니다. 다음은 관리형 DB 전환과 AI 고도화(실 블러·RAG·챗봇)로 파일럿 운영 수준까지 끌어올리는 것. 거기서 또 글로 만나요. 👋
