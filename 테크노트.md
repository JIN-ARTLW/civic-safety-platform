# 2주차 테크노트 — Claude Code 활용 AI 주도 웹 개발 파이프라인 (Spec-Kit)

사전 직무 교육 때 배운 **Claude Code 기반 AI 주도 웹 개발 파이프라인**을, 실제 프로젝트(*시민 참여형 스마트시티 안전 신고 플랫폼*)로 한 번 더 따라 하면서 정리했다. 앞으로 어떤 프로젝트든 이 흐름대로 가면 된다.

- **준비물**: 컴퓨터(여기선 mac 기준), Claude Code를 쓸 수 있는 계정
- **핵심 도구**: GitHub **Spec Kit** — 캔버스 → 명세 → 계획 → 작업 → 구현을 명령어로 진행
- **이번 결과물**: 사진 한 장으로 도시 위험을 신고하면 AI가 분류하고 안전신문고식 신고서를 자동 작성하는 웹서비스. GitHub Pages + cloudflared 터널로 외부 공개 완료.

> 💡 명령어 표기 주의: 교육 자료는 `/speckit.specify`(점)로 나오는데, 설치 버전에 따라 `/speckit-specify`(하이픈)일 수 있다. 안 먹히면 반대 표기로 시도하면 됨. 아래는 교육 자료 표기(점)를 따른다.

공통 패턴은 **`명령어` — `참조 .md 파일` — `구체적인 지시`** 세 덩어리로 요청하는 것이다.

---

## 1. Canvas(캔버스) 작성

캔버스란 사업/연구 아이디어를 **최대한 명확하고 구체적으로** 정의서 형태로 구체화한 것이다. 우리가 쓴 건 데이터 주도 비즈니스 모델 캔버스로, **개요 + 11칸**으로 구성된다.

```
개요
미션 / 핵심 활동 / 핵심 파트너 / 핵심 장벽 / 핵심 이네이블러 / 핵심 데이터
가치 제안 / 부정적 영향 / 혜택 / 비용 / 수익
```

각 칸을 최대한 구체적으로 채우는 게 중요하다. 주제에 따라 캔버스 틀은 달라질 수 있으니 자기 주제에 맞는 틀을 찾는 게 먼저다.

- 개인적으로는 노션에 작성 → `.md`로 저장 → 프로젝트 디렉토리에 둔다.
- 이번 프로젝트는 `business_model_canvas.md`로 저장했다. (이미지 캔버스를 .md로 옮겨 적는 것부터 시작)

> 이후 모든 단계가 이 캔버스(파일)를 근거로 돌아간다. 여기가 부실하면 뒤가 다 흔들린다.

---

## 2. 서비스 명세 구체화 — `/speckit.specify`

캔버스를 토대로 상세 서비스 명세를 생성하는 단계. `명령어 - 캔버스파일 - 지시`로 요청한다.

```
/speckit.specify 'business_model_canvas.md' 를 토대로 프로젝트 서비스 명세(Specification)를
상세하게 생성합니다. 논리적이고 체계적인 웹 기반 서비스 생성이 가능하도록 생성합니다.
```

→ 실행하면 **`spec.md`** 가 생성된다. (사용자 스토리, 기능 요구사항(FR), 측정 가능한 성공 기준(SC) 등) 우리 경우 사용자 스토리 4개·FR 29개·SC 11개가 나왔다.

---

## 3. 명세 오류 검토 및 수정 — `/speckit.clarify`

생성된 명세에 논리적 오류·누락이 없는지 재검토하고 수정하는 디버깅 단계. **명령어만 실행하면** 애매한 부분을 질문으로 던져준다.

```
/speckit.clarify
```

우리는 4가지를 여기서 못 박았다: **익명 vs 로그인(하이브리드로 결정)**, 멀티테넌시(지자체별 격리), 가용성 목표, 데이터 보관기간. 코드 짜기 전에 이걸 확정해두면 나중이 편하다.

## 4. 프로젝트 세부 계획 — `/speckit.plan`

검증된 명세로 **어떤 기술·툴로 구현할지** 빌드 계획을 세운다. 여기서 외부 리소스를 적극 참조·바인딩하는 게 핵심.

```
/speckit.plan 수정된 명세를 기반으로 구체적인 빌드 계획을 세웁니다.
아래 외부 마켓플레이스 및 UI 리소스를 적극 참조하여 계획에 바인딩합니다.
```

→ **`plan.md`** 생성. 이 단계에서 참고하면 좋은 곳:

| 용도 | 사이트 | 사용법 |
|------|--------|--------|
| 기능·스킬 스택 | **https://www.skills.sh** | 원하는 스킬 경로를 복사해 .md에 넣고 *"~관련 스킬이다, 설치하여 구현 시 참조한다"* 적기 |
| 스킬 마켓 보강 | **https://claudemarketplaces.com** | skills.sh로 부족할 때 |
| UI/UX 디자인 테마 | **https://getdesign.md** | 원하는 스타일의 설치 코드/프리뷰 복사 → *"이 스타일로 전체 일관되게 구성"* 첨언 |

우리는 supabase·shadcn·docx·pdf 등 스킬과, 관제=IBM Carbon / 시민=Intercom 디자인 테마를 plan에 바인딩했다.

## 5. 구현 task 분할 — `/speckit.tasks`

구현 직전, 작업을 체계적으로 분할한다. 이때 **[프론트엔드 · 백엔드 · 도메인] 3대 요소**가 명확히 반영됐는지 반드시 확인한다.

```
/speckit.tasks 계획한 작업들을 체계적으로 분할하고 준비합니다.
[프론트엔드, 백엔드, 도메인] 3대 요소가 명확히 반영됐는지 확인합니다.
```

→ **`tasks.md`** 생성(우리는 78개 task, 각 task에 FE/BE/DOM 라벨). 

> 팁: 아직 "어떻게 웹에 올릴지"가 안 정해졌으면 이때 Claude와 토의하면 된다. **목적·사용량 기준으로 추천**해준다 — 프로토타입이면 로컬, 소수면 터널, 100명 이상이면 클라우드 등.

## 6. 서비스 구현 — `/speckit.implement`

가이드 파일들을 참조해 실제 코드를 구현한다. **중요: 외부에서 바로 결과를 볼 수 있게 포트·도메인을 명확히 지정**해야 한다.

```
/speckit.implement 디렉토리 내 모든 파일을 참조하여 구현하라.
웹서비스 front-end, backend는 각각 9503, 9523이다. 구현할 도메인은 p3.sumzip.com 이다.
지정한 포트와 도메인으로 접근 가능하도록 보장하라.
```

→ 실행되면 실제 앱이 뜬다. 확인 후 수정이 필요하면 **원하는 바를 최대한 구체적으로** 말하면 째깍 고쳐준다.

> AI 주도 코딩의 핵심은 **내가 코드를 직접 안 건드리는 것**. 대신 "내가 결과물을 바로 확인"하는 게 중요하다. 그래서 포트·도메인 지정이 그렇게 중요하다.

### 실전 메모 — AI 분류 세 번 갈아엎은 이야기 🔄
구현하면서 제일 삽질한 부분. 처음엔 파일명 키워드(가짜) → 브라우저 객체탐지(COCO-SSD)로 갔는데, **"누가 봐도 포트홀"인 사진을 "객체 없음"**으로 처리했다. COCO는 사람·차 같은 일반 사물만 알지 '포트홀'은 모른다. 그래서 **제로샷 분류(CLIP)** 로 바꿔, 원하는 위험을 *문장으로 정의*해 사진과 의미 대조하게 했더니 포트홀·침수·쓰레기·불법주정차를 구분했다. (키·서버 없이 브라우저에서 동작) → *교훈: "사물을 탐지"하는 모델과 "상황을 이해"하는 모델은 다르다.*

## 7. 서비스 정상화 & 자가 루프 — `/loop`

도메인 접속 시 화면이 안 뜨는 등을 막기 위해, **누구나 접근 가능할 때까지** 시스템이 에러를 찾아 자동 보완하게 한다. 잘 되면 스킵 가능.

```
구현한 결과 서비스를 정상화하라. 누구나 접근 가능한지 테스트하라.
/loop 문제가 있다면 정상화할 때까지 개선하라.
```

> 실전 메모: 우리는 여기서 **502 Bad Gateway** 를 만났는데, 터널은 살아있고 그 뒤 게이트웨이가 죽어서 전달할 곳이 없던 거였다. 원인을 잡고 게이트웨이를 분리·재기동해 해결.

## 8. 배포 — 외부에 공개하기 (터널 / GitHub Pages)

> 교육 자료엔 짧게 나오지만, 실제로 제일 손이 가는 부분이라 따로 정리한다.

도메인이 내 소유가 아니면(우리처럼 `p3.sumzip.com`이 남의 장비를 가리키면) 그 도메인은 못 쓴다. 그럴 때 **계정·도메인 없이** 공개하는 두 방법:

### ① GitHub Pages — 상시·정적
프론트를 정적 버전으로 만들어 `docs/` 폴더에 두고 Pages를 켜면, **항상 켜진 영구 공개 URL**이 생긴다. 백엔드가 필요하면 로직을 브라우저로 포팅(localStorage)해 키·서버 없이 동작시킬 수 있다.
- 결과: `https://<아이디>.github.io/<레포>/`

### ② cloudflared 터널 — 풀스택·공유 백엔드
**내 컴퓨터를 그대로 외부 공개**한다. 계정도 도메인도 필요 없고 명령 한 줄.

```bash
brew install cloudflared
cloudflared tunnel --url http://localhost:9500   # 공개 URL 자동 발급(*.trycloudflare.com)
```

- 여러 사용자가 **같은 서버 데이터를 공유**하는 진짜 풀스택 데모가 된다.
- ⚠️ 단점: 켤 때마다 **URL이 바뀐다.** 맥이 켜져 있어야 한다. 고정 주소가 필요하면 Cloudflare 계정 연결(Named Tunnel)이나 클라우드 배포로 가야 한다.
- 팁: 프론트(:9503)와 백엔드(:9523)를 한 URL로 묶으려면, **게이트웨이**(정적 서빙 + `/api`는 백엔드로 프록시)를 하나 두고 그 포트(:9500)만 터널로 노출하면 깔끔하다.

> 어떤 방식이 맞는지 모르겠으면 Claude에게 "사용량·목적"을 말하고 물어보면 추천해준다.

## 9. 버전 관리 & Git push

중간중간 해도 되지만, 코드를 깃에 올리고 싶을 때 레포명 주고 push하게 하면 라벨링까지 알아서 해준다.

```
이 프로젝트를 내 깃허브에 올려줘. (필요 시 레포명/공개여부 지정)
```

→ 커밋 메시지·구조 다 잡아서 push. 이후 깃으로 버전 관리.

### 버전 백업(스냅샷) — 발표/마일스톤 직전 추천
"지금 잘 되는 이 상태"를 박제하려면 **태그 + 백업 브랜치**:

```bash
git tag -a v0.1-midterm -m "중간발표 버전 백업"
git branch backup/midterm-2026-06-19
git push origin v0.1-midterm
git push origin backup/midterm-2026-06-19
```

우리는 한 발 더 나가, 발표용으로 백업 버전의 정적 파일을 `docs/backup-v0.1/` 폴더에 **고정 복사**해서, 메인을 계속 고쳐도 *"발표 시점 그대로 볼 수 있는 페이지"* 를 따로 뒀다. 라이브 개발 링크 2개(Pages·터널)는 그대로 유지.

## 10. (옵션) 서버 관리 킬스위치 스크립트

배포 후 유지보수용으로, **프론트·백엔드·(터널) 상태를 확인하고 제어**하는 셸 스크립트를 만든다. 생성되면 `ls`로 파일 존재만 확인하면 됨. **(이 스크립트류·런타임 파일은 깃 커밋에서 빼는 게 좋다 — `.gitignore` 처리)**

```
현재 정상화된 서비스의 시작/정지/재시작을 셸 스크립트로 간결하게 만들어줘.
반드시 frontend, backend, (터널) 상태를 확인해야 하고, 옵션은 on/off/status/restart.
```

→ 우리 경우 `tunnel.sh`가 그 역할:

```bash
./tunnel.sh on       # 백엔드+게이트웨이 점검 후 터널 켜기(공개)
./tunnel.sh off      # 터널만 끄기(외부 차단)
./tunnel.sh status   # 상태 한눈에
./tunnel.sh restart  # 껐다 켜기(새 URL)
```

## 11. 재사용 레시피 — 위치 입력 & 키 안전 (이번에 추가로 익힌 것)

### 위치 입력 3단 (키 최소·무료)
1. **사진 EXIF 자동** — `exifr`(CDN)로 사진의 GPS·촬영시각을 자동 추출. `exifr.gps(file)`, `exifr.parse(file).DateTimeOriginal`.
2. **현재 위치** — `navigator.geolocation`.
3. **한국 주소 검색** — 다음(카카오) 우편번호 팝업 → 무료 지오코딩(Nominatim)으로 좌표 변환.
```html
<!-- 폐기된 index.v1 말고 v2 사용 (구URL은 403) -->
<script src="https://t1.daumcdn.net/mapjsapi/bundle/postcode/prod/postcode.v2.js"></script>
```
```js
new daum.Postcode({ oncomplete: d => geocode(d.roadAddress || d.jibunAddress) }).open();
// geocode: fetch('https://nominatim.openstreetmap.org/search?format=json&countrycodes=kr&q='+addr)
```
> 화면엔 위/경도 숫자칸을 숨기고(hidden) 위 3가지로만 채우면 UX가 깔끔하다. 좌표는 내부 값으로만 보관.

### 분류 체계 = 단일 소스 → 양식 자동 생성
- 위험 분류를 **한 곳(코드)** 에 정의(예: 안전신문고 3대 분야·세부유형)하고, 프론트/백엔드/AI 라벨/신고 양식이 **모두 그걸 참조**하게 한다. 유형을 늘리려면 그 한 곳만 고치면 끝.

### API 키는 비밀파일로 (절대 채팅·깃 금지)
- 서버가 `.run/anthropic.key`(`.gitignore`)에서 키를 읽게 하고, **키 없으면 무료 브라우저 AI로 폴백**되게 설계.
- ⚠️ **Claude Max 구독 ≠ API 키.** API 호출용 키는 `console.anthropic.com`에서 별도 발급·과금. 사진 익명 신고라면 민감정보를 아예 안 받는 게 정석.

## 12. 공공 서비스 전환 체크리스트 (보안·제도)

MVP를 실제 공공 서비스로 올릴 때 기능보다 **인증·보안·연동**이 관문이다.

- [ ] **본인확인은 직접 구현 말고 위임** — 간편인증·PASS(인증기관이 안전키패드 처리). 주민번호 미수집.
- [ ] **서버 공격 방어** — 웹방화벽(WAF) + DDoS 차단 + 속도제한 + **KISA 사이버대피소(무료)**.
- [ ] **개인정보** — 익명·최소수집·비식별(얼굴/번호판 블러)·암호화·자동파기 + **개인정보 영향평가(PIA)**.

## 정리 (한 장 요약)

1. **Canvas 작성** — 최대한 명확하고 구체적으로 (`.md`로 저장)
2. **`/speckit.specify`** — 캔버스 토대로 명세 구체화 → `spec.md`
3. **`/speckit.clarify`** — 명세 오류·누락 탐지 및 수정
4. **`/speckit.plan`** — 세부 계획 (skills.sh·claudemarketplaces·getdesign.md 바인딩) → `plan.md`
5. **`/speckit.tasks`** — 구현 task 분할 (**FE·BE·도메인 3요소** 확인) → `tasks.md`
6. **`/speckit.implement`** — 구현 (**포트·도메인 명확히 제공**) → 수정 반복
7. **정상화** — 누구나 접근 가능한지 테스트, `/loop`로 자동 보완
8. **배포** — GitHub Pages(상시·정적) / cloudflared **터널**(풀스택·공유)
9. **Git push** + 버전 백업(태그/브랜치/스냅샷 폴더)
10. **킬스위치 스크립트** (start/stop/status/restart, 깃 커밋 제외)

### 꼭 기억할 팁
- 캔버스를 **구체적으로**. 뒤 단계가 전부 여기서 파생된다.
- 매 단계 **`명령어 - 캔버스 기반 .md 파일 - 구체 지시`** 로 파일을 참조시킬 것.
- task 단계 참고: skills(https://www.skills.sh), 마켓(https://claudemarketplaces.com), 디자인(https://getdesign.md)
- **포트·도메인을 명확히** 줘야 결과를 바로 본다.
- AI 주도 코딩은 **내가 코드를 안 건드리고**, 결과를 바로 확인·구체적으로 피드백하는 게 핵심.

### 트러블슈팅 메모 (실제로 겪은 것)
- **Spec 명령 인식 안 됨** → 스킬은 세션 시작 시 로드됨. 설치 직후엔 재시작 후 사용(또는 SKILL.md 수동 실행).
- **AI가 포트홀을 "객체 없음"** → 객체탐지(COCO) 대신 **제로샷 분류(CLIP)** 로. "사물 탐지"와 "상황 이해"는 다른 모델.
- **주소 검색 모듈 로드 실패(403)** → 다음 우편번호 구URL 폐기. `postcode.v2.js`로 교체.
- **터널 502 Bad Gateway** → 터널은 살아있고 뒤의 게이트웨이/백엔드가 죽은 것. 같은 포트로 재기동하면 URL 유지.
- **무료 터널 URL이 매번 바뀜** → 마일스톤 페이지엔 하드코딩 말고 "발표 시 안내". 고정 주소는 Cloudflare 계정(Named Tunnel)·클라우드.
- **런타임/비밀 파일이 깃에 올라감** → `.run/`·업로드·키 파일은 `.gitignore`. 이미 올라갔으면 추적 해제.
