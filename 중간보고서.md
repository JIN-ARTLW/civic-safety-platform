## 1. 개요 (Executive Summary)

멀티모달 AI(이미지 분류)와 지리정보(GIS)를 결합하여, **시민이 촬영한 한 장의 사진으로 도시 위험 요소를 자동 식별·분류하고 지자체 행정과 연동**하는 B2G/B2C 하이브리드 스마트시티 안전 플랫폼이다.

본 중간보고 시점에서 **실제 동작하고 외부에서 접속 가능한 MVP**를 구축·배포 완료하였다. 핵심 흐름인 *"사진 촬영 → AI 위험 분류 → 안전신문고식 신고서 자동 작성 → 지자체 관제 대시보드 처리"* 가 키 없이도 브라우저 비전 모델(CLIP)로 실동작하며, 두 개의 공개 채널(상시 GitHub Pages, 풀스택 터널)로 시연 가능하다.

> **정직 고지**: 본 MVP는 기획서(plan.md)가 지정한 풀스택(Supabase/PostGIS/pgvector + Next.js + Python)을 검증 가능 환경 제약상 그대로 구동하기 어려워, **동일한 도메인 로직·API 계약·UX를 의존성 없는 Node + 브라우저 ML**로 구현한 것이다. 비즈니스 로직·워크플로우·UI·배포·이미지 AI 분류는 실동작하며, 실서비스 인프라(관리형 DB·실 LLM/RAG·얼굴 블러 등)는 후속 전환 대상이다.

## 2. 추진 배경 및 비즈니스 모델 (캔버스 기반)

### 2.1 문제의식 (미션)
기술적·행정적 장벽을 제거하여 시민 참여형 민원 생태계를 활성화하고, AI 기반 선제적 위험 분류로 지자체 행정 공백을 최소화하여 가장 안전한 도시 환경을 구현한다.

### 2.2 핵심 가치제안 (Value Proposition)
- **시민 관점**: "텍스트 타이핑 없는 3초 민원" — 사진 한 장으로 신고서가 자동 작성되고, 진행 상황을 대화형으로 안내.
- **지자체 관점**: "민원 분류·처리 시간 단축" — AI 자동 분류·우선순위·중복 묶음으로 행정 효율 제고.

### 2.3 비즈니스 모델 캔버스 요약
| 구분 | 내용 |
|------|------|
| 핵심 활동 | 모바일 웹 사진 신고 접수, AI 객체/위험 분류, 공문서 자동 생성, 대화형 안내 |
| 핵심 파트너 | 지자체(구청/시청) 유관 부서, 국토정보·공공데이터 플랫폼, 안전신문고(연계 목표) |
| 핵심 데이터 | 시민 UGC(현장 사진·GPS), 공공/행정 데이터, RAG용 규정·양식 벡터 |
| 고객/수익 | B2G SaaS 구독, 정부 과제·바우처, B2B 비식별 데이터 리포트 |
| 핵심 장벽 | 야간/저화질 인식률, 초상권·개인정보(PII), 허위·중복 신고 |

### 2.4 안전신문고 체계와의 정합
실제 안전신문고 신고 양식(3대 분야)에 맞춰 분류·양식을 설계하였다.
- **자동차·교통 위반** (불법 주·정차, 교통법규, 번호판, 불법 튜닝)
- **생활불편** (쓰레기 무단투기, 불법 광고물, 자전거·이륜차 방치, 기타)
- **안전** (도로·시설물 파손, 여름철 침수, 대기/수질 오염, 소방안전, 기타)

## 5. 구현 현황 (FE / BE / 도메인 3대 요소)

### 5.1 완료 (실동작 ✅)
- **시민 신고(US1)**: 실제 사진 업로드 + 미리보기 → AI 분류 → 접수번호 발급 → 상태 조회
- **AI 위험 분류**: 브라우저 **CLIP 제로샷**(Transformers.js)로 사진 의미 분석, 안전신문고 3분야·14유형 분류 (키 불필요). 키 설정 시 **Claude 비전** 자동 승급(코드 완비). 노면 손상 휴리스틱 폴백.
- **섹션별 신고 양식 + AI 자동작성**: 분야 인식 → 해당 양식 표시, **제목·신고내용·세부유형·발생일시 자동 채움**(수정 가능), 교통 분야는 차량번호 필드 추가.
- **위치 입력**: ① 사진 **EXIF GPS 자동 추출** → ② 현재 위치 → ③ **다음(카카오) 우편번호 주소검색 + 무료 지오코딩(Nominatim)**. 일시는 EXIF 우선, 없으면 공란 입력.
- **지자체 관제(US2)**: 테넌트 격리 신고 목록(우선순위·중복 묶음), 상태 전이(접수→처리중→완료, 잘못된 전이 차단), 처리 이력, **공문서 초안 자동 생성**, 사진 썸네일·AI 요약·차량번호·일시 표시.
- **대화형 안내(US3)**: 접수번호 진행상황 근거 기반 안내(스텁), 근거 없으면 담당 창구 안내.
- **신뢰도·보상(US4)**: 위치·유형 기반 중복 묶음, 로그인 신고 보상 적립.
- **배포**: 상시 GitHub Pages(정적) + 풀스택 터널(cloudflared, `tunnel.sh` 킬스위치).

### 5.2 진행/스텁 (⚠️)
- AI 비식별(얼굴/번호판 블러), RAG/LLM 공문서·안내는 **동작 스텁**(인터페이스 동일, 실모델 미탑재).
- 데이터 저장은 파일(JSON)/localStorage로 Supabase/RLS를 대체.

### 5.3 미착수 (실서비스 전환 잔여)
Supabase/PostGIS/pgvector·RLS, Next.js 이관, 알림 발송(푸시/이메일), 보관·파기 배치, 테넌트 설정관리 UI, CI/부하/보안, 고정 도메인.

> 작업 진척: **체크리스트 34/78 완료** (기능 의도 기준). 상세는 [tasks.md](specs/001-civic-safety-platform/tasks.md) *Implementation Status*.

## 8. 검증 결과

- E2E(curl): 안전(도로파손→도로관리과)·교통(불법주정차+차량번호→교통지도과) 분류·라우팅·양식필드·AI요약 대시보드 전달 정상.
- 테넌트 격리: 타 지자체 토큰 조회 차단(403), 관할 외 좌표 안내.
- 상태 전이: 정상 전이 동작, 비정상 전이 422 차단.
- 위치: Nominatim 지오코딩 실좌표 반환, EXIF GPS 추출 동작.
- 배포: 양 채널 HTTP 200 및 콘텐츠 반영 확인.

## 10. 향후 계획 (로드맵)

1. **실 인프라 전환**: Supabase(PostgreSQL+PostGIS+pgvector) + RLS 멀티테넌시, Next.js 이관.
2. **AI 고도화**: 얼굴/번호판 실블러, RAG/LLM 공문서·대화 안내, (선택) Claude 비전 상시화.
3. **행정 연동**: 안전신문고/지자체 시스템 실연동, 알림 발송(푸시/이메일).
4. **운영화**: 보관·파기 배치, 테넌트 설정관리, CI/부하/보안, 고정 도메인·가용성 99.9%.
5. **확장**: 파일럿(청주) 검증 → 다수 지자체 온보딩(SaaS).

## 11. 결론

본 중간 단계에서 **비즈니스 모델(캔버스) → 명세 → 구현 → 공개 배포**까지의 일관된 파이프라인을 완성하고, 핵심 가치인 *"사진 한 장 신고 → AI 분류 → 안전신문고식 자동 신고서 → 지자체 처리"* 를 **실제 동작·시연 가능한 형태**로 입증하였다. 무료 브라우저 AI로 키·비용 없이 핵심 기능을 구현했고, 실서비스 인프라·실 LLM/RAG·개인정보 보호는 후속 단계로 명확히 구분하였다. 다음 단계는 관리형 DB 전환과 AI 고도화를 통해 파일럿 운영 수준으로 끌어올리는 것이다.
